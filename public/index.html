Toys API Documentation

HTTP Method Route Description Parameters / Request Body
GET /toys Returns A List Of Toys According To The query.
  • page (optional): Page Number To Display (default: 1), For Example:
    https://noams-toys-api-doc.cyclic.app/toys?page=1

  • perPage (optional): Toys To Display Per Page (default: 10, maximum: 10), For Example: https://noams-toys-api-doc.cyclic.app/toys?perPage=5

  • sort (optional): Query To Sort By (default: price), For Example:
    https://noams-toys-api-doc.cyclic.app/toys?sort=price

  • reverse (optional): If Reverse Is Set To yes, The Sort Will Be In A Descending Order, For Example:
    https://noams-toys-api-doc.cyclic.app/toys?reverse=yes
GET /toys/price Returns A List Of Toys That Has Been Filtered By The Desired Price Range.
  • min (optional): Minimum Price Filter (default: 0), For Example:
    https://noams-toys-api-doc.cyclic.app/toys/price?min=0

  • max (optional): Maximum Price Filter (default: maximum safe integer), For Example:
    https://noams-toys-api-doc.cyclic.app/toys/price?max=200
GET /toys/singleToy/:id Returns A Single Toy By Its ID.
  • id : Returns An Item By Its ID, For Example:
    https://noams-toys-api-doc.cyclic.app/toys/singleToy/63fddfb046d8eb4ddc31513a
GET /toys/search Returns A Toys List Matching A Query.
  • s : Query To Use In Order To Return Matched Toy Names Or Descriptions, For Example:
    https://noams-toys-api-doc.cyclic.app/toys/search?s=groot
GET /toys/category/:catName Returns A List By Category.
  • catName : The Name Of The Category That Will Return A Toy List Based On The Category, For Example: https://noams-toys-api-doc.cyclic.app/toys/category/lego
POST /toys Adds A New Toy.
  • Request Body : Toy Data To Be Added Has To Follow The Format Below.
  • img_url Attribute Is Allowed But Its Not Required. 

    • {
  "name": "string",
  "info": "string",
  "category": "string",
  "price": number,
  "img_url": "string"
}
PUT /toys/:idEdit Updates An Existing Toy.
  • idEdit : the ID of the toy to update.
  • Request Body : Toy Data To Be Added Has To Follow The Format Below.
  • img_url is allowed and not required 

    • {
  "name": "string",
  "info": "string",
  "category": "string",
  "price": number,
  "img_url": "string"
}
DELETE /toys/:idDel Deletes A Toy By Its ID.
  • idDel : ID Of The Toy To Delete.
